# 客服工单自动分配系统

## 1. 题目目标

设计并实现一个**客服工单分配器（Dispatcher）+ 模拟器**，用于在动态来单和人员波动条件下平衡：

- 紧急工单低等待、尽快被接单
- 客服总体忙闲均衡，提升处理效率
- 普通工单不过度积压，避免长期饥饿
- 技能匹配尽量合理，减少错误分配和重复转派

------

## 2. 场景设定

某公司有一个在线客服支持系统，需要将用户提交的工单自动分配给客服人员处理。

### 客服属性

系统包含 N 个客服（Agent 0 ~ Agent N-1），每个客服具有以下属性：

- agent_id
- skill_set（如 billing / login / api / deployment）
- shift_start、shift_end
- max_concurrent（最大同时处理工单数，MVP 中可设为 1 或 2）
- status（online / offline / busy）

### 工单属性

每个工单具有以下属性：

- ticket_id
- priority（P1 / P2 / P3）
- category（对应技能类型）
- created_time
- estimated_handle_time

其中：

- `estimated_handle_time` 与真实处理时长存在误差（约 ±30%）
- 工单到达是动态的，不同优先级到达模式不同

### 工单类型

- 紧急工单（P1）：全天随机提交，要求快速响应
- 普通工单（P2 / P3）：白天集中出现，允许一定排队

------

## 3. 运行规则（必做）

### 3.1 分配与处理

- 工单只有在分配给具备对应技能且有空闲容量的客服后，才能进入处理状态
- 客服处理期间会占用一个并发槽位
- 若无合适客服空闲，工单进入等待队列

### 3.2 技能匹配

- 工单优先分配给拥有对应 `category` 技能的客服
- 若长时间无匹配客服可用，可允许降级分配给“通用客服”（若实现）
- 错误技能分配会导致处理时间额外增加，例如增加 50%

### 3.3 转派机制

- 已分配但发现技能不匹配的工单，允许转派一次
- 被转派的工单会重新进入队列，并额外增加约 5 分钟处理成本
- 同一工单最多转派 1 次，避免无限循环

### 3.4 人员波动

- 客服可能临时离线（如休息、掉线、请假）
- 若客服离线时正在处理工单，则：
  - 该工单回到队列等待重新分配
  - 已消耗时间不清零，但额外增加约 10 分钟上下文切换成本

### 3.5 SLA 规则

系统至少需要考虑以下服务目标：

- P1 工单：从创建到首次被分配，目标不超过 5 分钟
- P2 工单：目标不超过 30 分钟
- P3 工单：目标不超过 120 分钟

------

## 4. 输出与指标要求（必做）

分配器应输出可复盘日志，并至少统计以下定量指标：

### 4.1. 紧急工单响应指标

- P1 工单平均等待时长
- P1 工单 P95 等待时长
- P1 工单 SLA 达成率（5 分钟内被分配的比例）

### 4.2. 总体处理效率指标

- 全部工单平均等待时长
- 全部工单平均完成时长
- 工单队列平均长度
- 工单转派率

### 4.3. 人员利用率指标

- 客服平均利用率
- 客服利用率方差
- 高峰时段系统吞吐量（每小时处理工单数）

### 4.4. 公平性指标

- 每位客服处理工单数的最大值、最小值、方差
- 最长未被处理普通工单等待时长
- 饥饿工单数（如等待超过其 SLA 2 倍的工单数量）

### 4.5. 错误分配指标

- 技能不匹配分配次数
- 因转派带来的额外处理成本总时长

------

## 5. 测试与验收要求（必做）

至少完成以下测试：

### 5.1. 基线对比

与至少 1 个基线策略对比，例如：

- FIFO（先来先分）
- 纯优先级分配（只看 P1/P2/P3，不看均衡）
- 最空闲客服优先

### 5.2. 高峰场景

模拟白天某时段普通工单集中涌入，同时穿插少量高优 P1 工单，观察：

- P1 是否仍能快速被响应
- 普通工单是否大面积积压

### 5.3. 离线场景

模拟部分客服中途离线，观察：

- 正在处理的工单是否被正确回收
- 系统能否继续稳定分配
- SLA 指标是否显著恶化

### 5.4. 转派场景

模拟工单技能标签与客服技能不完全匹配的情况，观察：

- 错误分配次数
- 转派率
- 转派对整体效率的影响

### 5.5. 结果可复现

- 同一随机种子下，模拟结果应一致
- 日志与指标统计可重复生成

## 7. 加分项

- 自适应策略：根据队列长度、SLA 违约率动态调整分配规则
- 更细粒度的技能模型：支持主技能、辅技能
- 多目标优化：在 SLA、利用率、公平性之间动态平衡
- 可插拔策略框架：支持快速替换不同分配算法
