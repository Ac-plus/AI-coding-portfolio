# 运行指南

本文档说明当前项目的运行方式，包括环境准备、单场景仿真、策略切换、总测试执行、测试报告生成以及常见输出文件位置。

## 1. 项目概览

当前项目已经实现以下能力：

- 加载场景配置并运行客服工单分配仿真
- 支持多种分配策略切换
- 输出仿真结果、日志和指标统计
- 执行单元测试与集成测试
- 生成覆盖 README 要求的测试报告

## 2. 运行环境

### 2.1 Python 版本

建议使用：

- `Python 3.11` 及以上

### 2.2 依赖说明

当前版本使用 Python 标准库即可运行，不依赖额外第三方包。

如果你想确认当前环境可用，可以在项目根目录执行：

```bash
python --version
```

## 3. 项目根目录

所有命令都默认在项目根目录执行，也就是当前目录：

```bash
/root/ticket_assign
```

## 4. 主要目录与文件

### 4.1 核心入口

- [main.py](/root/ticket_assign/src/ticket_assign/main.py)

  总测试的结构化原始结果

## 5. 单场景运行

### 5.1 基本命令

使用主程序运行一个场景：

```bash
PYTHONPATH=src python -m ticket_assign.main scenarios/minimal.json --pretty
```

参数说明：

- `scenarios/minimal.json`
  表示要加载的场景文件

- `--pretty`
  让输出结果以格式化 JSON 展示，便于阅读

如果不加 `--pretty`，程序会输出一行紧凑 JSON。

### 5.2 使用脚本运行

你也可以用脚本方式运行：

```bash
PYTHONPATH=src python scripts/run_simulation.py scenarios/minimal.json --pretty
```

这和直接调用 `python -m ticket_assign.main` 是等价的。

## 6. 切换分配策略

当前支持的策略有：

- `fifo`
- `priority_first`
- `least_loaded`
- `balanced`

### 6.1 指定策略运行

例如，使用 `balanced` 策略运行高峰场景：

```bash
PYTHONPATH=src python -m ticket_assign.main scenarios/peak_load.json --dispatcher balanced --pretty
```

例如，使用 `priority_first` 策略运行优先级对比场景：

```bash
PYTHONPATH=src python -m ticket_assign.main scenarios/priority_compare.json --dispatcher priority_first --pretty
```

### 6.2 参数说明

- `--dispatcher fifo`
  先来先分

- `--dispatcher priority_first`
  纯优先级分配

- `--dispatcher least_loaded`
  最空闲客服优先

- `--dispatcher balanced`
  综合考虑优先级、等待时长和均衡性的主策略

## 7. 仿真输出说明

单次仿真会输出一个结果对象，主要包含：

- `scenario_name`
  场景名称

- `dispatcher`
  使用的策略

- `final_time`
  仿真结束时间

- `queue_size`
  仿真结束时队列长度

- `agents`
  客服摘要信息，包括最终状态、最大并发和处理工单数

- `tickets`
  工单摘要信息，包括创建时间、首次分配时间、完成时间、转派次数等

- `logs`
  可复盘日志

- `metrics`
  指标统计结果，覆盖 README 第 4 节要求的响应、效率、利用率、公平性、错误分配指标

## 8. 运行单元测试

项目中的单元测试位于 [tests/unit](/root/ticket_assign/tests/unit)。

执行方式：

```bash
PYTHONPATH=src python -m unittest discover -s tests/unit -v
```

当前单元测试主要覆盖：

- 场景加载
- 最小闭环流程
- 离线回收
- 转派流程
- 多策略行为差异
- 指标统计

## 9. 运行集成测试

项目中的集成测试位于 [tests/integration](/root/ticket_assign/tests/integration)。

执行方式：

```bash
PYTHONPATH=src python -m unittest discover -s tests/integration -v
```

当前集成测试主要验证：

- 总测试脚本可以成功执行
- 主目录测试报告可以成功生成
- 报告内容覆盖 README 要求的测试项和指标项

## 10. 生成总测试报告

如果你想一次性执行 README 第 5 节要求的测试，并生成测试报告，执行：

```bash
PYTHONPATH=src python scripts/run_benchmark.py
```

执行完成后会生成：

- [测试报告.md](/root/ticket_assign/测试报告.md)
- [benchmark_results.json](/root/ticket_assign/outputs/metrics/benchmark_results.json)

### 10.1 该脚本会执行哪些测试

该脚本会自动执行以下测试：

1. 基线对比
   包括 `FIFO`、`PriorityFirst`、`LeastLoaded`、`BalancedDispatcher`

2. 高峰场景测试
   验证普通工单集中涌入时 P1 响应和普通工单积压情况

3. 离线场景测试
   验证客服离线时工单是否被正确回收并恢复处理

4. 转派场景测试
   验证技能不匹配分配、转派率和额外成本

5. 可复现测试
   验证相同随机种子下日志与指标是否一致

## 12. 常见问题

### 12.1 为什么命令前要加 `PYTHONPATH=src`

因为源码采用 `src/` 布局，设置 `PYTHONPATH=src` 后，Python 才能正确找到 `ticket_assign` 包。

### 12.2 如果提示找不到模块怎么办

请先确认：

- 当前工作目录是项目根目录 `/root/ticket_assign`
- 命令前已经加上 `PYTHONPATH=src`

### 12.3 如果只想看策略结果，不想跑总测试怎么办

直接运行单场景命令即可，例如：

```bash
PYTHONPATH=src python -m ticket_assign.main scenarios/peak_load.json --dispatcher balanced --pretty
```

### 12.4 如果想重新生成测试报告怎么办

重新执行：

```bash
PYTHONPATH=src python scripts/run_benchmark.py
```

新报告会覆盖主目录下的 [测试报告.md](/root/ticket_assign/测试报告.md)。
