# 会议室预订系统

## 原始开发需求

实现一个**包含前后端的**简单会议室预定系统。必须实现的功能如下：

1. 基础功能：实现预定、取消功能
2. 需要实现基础数据结构
3. 正确处理并发场景：系统需要支持多个用户同时预定
4. 正确处理冲突：确保多用户同时预定同一时间段的正确应对
5. 正确实现死锁防止策略
6. 各团队有一些不同的些业务规则，需要设计实现，例如：
   - 单次预定不超过4小时
   - 同一用户每天最多3个时段
   - 不能预定过去时间
   - 必须提前15分钟预定
   - 过期30分钟自动释放

边界如下：

1. 暂不需要实现数据库。可以用json文件代替。
