# 会议室预订系统运行指南

## 1. 说明

本项目是一个前后端分离的会议室预订系统：
- 后端：FastAPI
- 前端：React + Vite
- 数据存储：JSON 文件

当前代码目录中的关键部分如下：

```text
app/        后端代码
frontend/   前端代码
data/       JSON 数据文件
tests/      自动化测试
```

---

## 2. 环境要求

建议使用以下环境：
- Python：3.13
- Conda 环境：`meeting`
- Node.js：用于运行前端

本项目的后端测试要求统一在 `meeting` 环境中执行。

---

## 3. 后端运行

### 3.1 激活环境

```bash
conda activate meeting
```

如果你的 shell 中无法直接使用 `conda activate`，也可以使用：

```bash
/root/miniforge3/bin/conda activate meeting
```

### 3.2 安装后端依赖

如果 `meeting` 环境里还没有安装依赖，可以执行：

```bash
pip install -r requirements.txt
```

### 3.3 启动后端服务

在项目根目录执行：

```bash
uvicorn app.main:app --reload
```

默认启动地址：

```text
http://127.0.0.1:8000
```

接口文档地址：

```text
http://127.0.0.1:8000/docs
```

健康检查接口：

```text
GET /health
```

---

## 4. 前端运行

### 4.1 进入前端目录

```bash
cd frontend
```

### 4.2 安装前端依赖

```bash
npm install
```

### 4.3 启动前端开发服务

```bash
npm run dev
```

Vite 默认地址通常为：

```text
http://127.0.0.1:5173
```

### 4.4 前后端联调说明

当前前端代码默认直接请求：

```text
/api/rooms
/api/bookings
/api/maintenance/release-expired
```

因此开发时通常有两种方式：

1. 让前端开发服务器配置代理到后端 `8000`
2. 或者把前端产物与后端放在同域下部署

如果没有代理，浏览器中直接运行前端时，请求可能无法正确转发到后端。

---

## 5. 数据文件说明

系统当前使用 JSON 文件存储数据：

```text
data/rooms.json
data/bookings.json
```

说明：
- `rooms.json`：会议室初始数据
- `bookings.json`：预订记录

如果你希望重置测试数据，可以手动编辑这些文件。

例如，将 `bookings.json` 重置为空：

```json
[]
```

---

## 6. 自动释放说明

系统支持“过期 30 分钟自动释放”：
- 应用启动后，会自动启动后台释放任务
- 后台任务会周期性扫描过期预订并将其标记为 `expired`
- 同时系统也提供手动触发接口：

```text
POST /api/maintenance/release-expired
```

前端页面中也提供了“手动释放过期预订”按钮，便于联调和演示。

## 8. 常用接口一览

### 8.1 查询会议室

```http
GET /api/rooms
```

### 8.2 查询预订

```http
GET /api/bookings
```

支持可选查询参数：
- `user_id`
- `room_id`
- `date`
- `include_inactive`

### 8.3 创建预订

```http
POST /api/bookings
Content-Type: application/json
```

示例请求体：

```json
{
  "room_id": "room-a",
  "user_id": "user-1",
  "start_time": "2026-04-02T10:00:00",
  "end_time": "2026-04-02T11:00:00"
}
```

### 8.4 取消预订

```http
DELETE /api/bookings/{booking_id}?user_id=user-1
```

### 8.5 手动释放过期预订

```http
POST /api/maintenance/release-expired
```

## 9. 常见问题

### 9.1 后端能启动，但前端打不开

请先确认：
- 是否已经安装 Node.js
- 是否执行了 `npm install`
- 是否执行了 `npm run dev`

### 9.2 前端页面打开了，但接口请求失败

通常是因为：
- 后端服务未启动
- 前端未配置代理
- 前后端不在同一域名/端口下

建议先确认后端 `http://127.0.0.1:8000/docs` 是否可访问。

### 9.3 如何清空已有预订数据

直接把：

```text
data/bookings.json
```

改成：

```json
[]
```

### 9.4 如何确认当前系统是否正常

最简单的方法：

1. 启动后端
2. 访问 `/docs`
3. 调用 `GET /api/rooms`
4. 再执行测试：

```bash
/root/miniforge3/envs/meeting/bin/pytest -q
```

## 10. 当前已知限制

1. 当前数据存储是 JSON 文件，不是数据库
2. 当前并发控制基于单进程内锁，适合本项目当前规模
3. 当前前端是否能本地构建，取决于机器上是否安装了 Node.js
4. 当前自动释放任务已实现，但周期参数仍是代码中的固定配置

这些限制不影响当前 README 中的功能需求完成情况。
