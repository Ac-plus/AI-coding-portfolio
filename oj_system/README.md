# 轻量级OJ系统开发需求

## 功能需求

实现一个轻量版的在线代码提交与运行（OJ）系统，应当具有以下功能：

- 可以在题库中添加题目，首页进去可以浏览并进入题目，并且作答。
- 题目需要有“分类”，例如“动态规划”“哈希表”，等等。一个题可以同时属于多个类。这就需要你在首页实现用于标识题目分类的页签，可以点击切换。
- 对于每道题目：
  - 包含题目描述、输入格式、输出格式、提示这几个部分；
  - 用户可以提交代码，并进行在线评分。原理是系统通过在线测试用例的评测，给出用户的分数（通过个数/总个数）
  - 要像力扣一样，使用`class Solution{int your_func(int val) {}};`这种 类+函数 的格式作答，评测系统也要支持这样的测试
  - 评测时，需要仿照力扣，对可能出现的错误（解答错误、超时、编译错误等）进行判断并且显示给用户
- 交互界面仿照力扣的格式，左边题目、右边代码作答区域
- 需要实现前后端，也就是交付后用户可以在网页上使用

## 边界

- 暂不需要实现数据库，可用文件形式存储每道题目、测试用例等数据
- 暂不需要实现多用户登录、鉴权等逻辑
- 系统只需要支持用户用C++语言即可
