# 运行指南

本文档用于帮助你正式启动和使用当前的轻量级 OJ 系统。

## 1. 运行前准备

当前项目约定所有环境相关操作都在 `oj` conda 环境中进行。

先确认环境存在：

```bash
conda env list
```

如果输出中包含 `oj`，即可继续。

## 2. 启动服务

在项目根目录 `/root/oj_system` 下执行：

```bash
cd /root/oj_system
PYTHONPATH=/root/oj_system conda run --no-capture-output -n oj python3 scripts/run_backend.py
```

如果启动成功，终端会看到类似输出：

```text
OJ backend listening on http://0.0.0.0:8000
```

此时服务已经启动，默认访问地址为：

```text
http://127.0.0.1:8000
```

停止服务时，在当前终端按 `Ctrl + C` 即可。

## 3. 页面使用说明

启动服务后，在浏览器访问：

```text
http://127.0.0.1:8000
```

页面分为左右两栏：

- 左侧：题目分类、题目列表、题目录入区域
- 右侧：题目详情、代码编辑区、提交结果区

### 3.1 浏览题目

- 左上角会显示系统标题和“新增题目”按钮。
- 分类区域会展示已有标签，例如“哈希表”“数组”。
- 点击某个标签后，题目列表会按该分类筛选。
- 点击“查看全部”可以恢复完整题库。
- 点击题目列表中的任意题目，可以在右侧查看题面详情。

### 3.2 阅读题目

题目详情区域默认展示：

- 题目描述
- 输入格式
- 输出格式
- 提示
- 函数签名
- 测试数据预览

当前示例题为 `two-sum`，你可以直接用它进行首次验证。

### 3.3 编写代码并提交

右侧代码编辑区会自动填入该题的起始模板。当前系统要求按如下风格提交 C++ 代码：

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        // your code
    }
};
```

提交方式：

1. 在右侧编辑器中修改代码。
2. 点击“提交评测”按钮。
3. 等待判题完成。

提交结果区域会显示：

- 判题状态
- 通过数量 / 总用例数
- 分数
- 编译输出（如果有）
- 每个测试用例的期望结果与实际结果

当前系统支持显示这些典型状态：

- `Accepted`
- `Wrong Answer`
- `Compile Error`
- `Runtime Error`
- `Time Limit Exceeded`

## 4. 新增题目说明

点击左上角“新增题目”按钮后，会展开题目录入表单。

你需要填写：

- 题目 ID
- 标题
- 难度
- 标签
- 描述
- 输入格式
- 输出格式
- 提示
- 函数名
- 返回类型
- 参数 JSON
- 测试用例 JSON
- 时间限制
- 内存限制
- 起始代码

其中两个最关键字段是：

### 4.1 参数 JSON

示例：

```json
[
  { "name": "nums", "type": "vector<int>" },
  { "name": "target", "type": "int" }
]
```

### 4.2 测试用例 JSON

示例：

```json
[
  { "input": [[2, 7, 11, 15], 9], "expected": [0, 1] },
  { "input": [[3, 2, 4], 6], "expected": [1, 2] }
]
```

填写完成后点击“保存题目”，题目会写入本地文件题库，并自动出现在题目列表中。

## 5. 数据存储位置

系统当前不使用数据库，数据都以文件方式保存。

- 题目数据目录：`data/problems/`
- 提交记录目录：`data/submissions/`
- 判题临时目录：`backend/temp/`

说明：

- 新增题目后，会在 `data/problems/` 下生成对应的 `.json` 文件。
- 每次提交后，会在 `data/submissions/` 下生成一条提交记录。
- 评测时生成的中间编译文件会放在 `backend/temp/`。

## 7. 接口说明

当前系统提供这些主要接口：

- `GET /api/health`：检查服务是否正常
- `GET /api/problems`：获取题目列表
- `GET /api/problems/{id}`：获取单题详情
- `GET /api/tags`：获取全部标签
- `POST /api/problems`：新增题目
- `POST /api/submissions`：提交代码并判题
- `GET /api/submissions/{id}`：查询某次提交结果

### 7.1 提交代码请求示例

```json
{
  "problem_id": "two-sum",
  "source_code": "#include <vector>\n#include <unordered_map>\nusing namespace std;\n\nclass Solution {\npublic:\n    vector<int> twoSum(vector<int>& nums, int target) {\n        unordered_map<int, int> seen;\n        for (int i = 0; i < (int)nums.size(); ++i) {\n            int need = target - nums[i];\n            if (seen.count(need)) return {seen[need], i};\n            seen[nums[i]] = i;\n        }\n        return {};\n    }\n};"
}
```

## 8. 常见问题

### 8.1 页面打不开

先检查服务是否启动成功：

```bash
curl http://127.0.0.1:8000/api/health
```

如果返回：

```json
{"status": "ok"}
```

说明后端已经启动。

### 8.2 提交时报编译错误

请确认：

- 代码语言是 C++
- 提交内容中包含 `class Solution`
- 函数名与题目要求一致
- 参数列表与题面函数签名一致

### 8.3 新增题目失败

常见原因有：

- 题目 ID 已存在
- `parameters` 不是合法 JSON
- `test_cases` 不是合法 JSON
- 某些必填字段为空

### 8.4 为什么没有登录功能

这是当前版本的设计边界之一。当前系统只实现单机可用的轻量版 OJ，不包含用户登录、鉴权和数据库。
